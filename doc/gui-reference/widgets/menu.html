Element <menu>

Description

Specifies a menu. A menu is a widget which can be selected and focused.

Syntax

<menu
	[name=<string>]
	[size=<size>]
	[class=<string>]
	[bgcolor=<color>] | [[bgcolor.r=<red>] [bgcolor.g=<green>] [bgcolor.b=<blue>] [bgcolor.a=<alpha>]]
	[selbgcolor=<color>] | [[selbgcolor.r=<red>] [selbgcolor.g=<green>] [selbgcolor.b=<blue>] [selbgcolor.a=<alpha>]]
	[bgimage=<file>] | [[bgimage.path=<path>] [bgimage.name=<filename>]]
	[selbgimage=<file>] | [[selbgimage.path=<path>] [selbgimage.name=<filename>]]
	[margin=<pixel>]
	[focusable=<bool>]
	[selectable=<bool>]
	[up_arrow=<widget name>]
	[down_arrow=<widget name>]
	[left_arrow=<widget name>]
	[right_arrow=<widget name>]
	[data=<data string>]
	[navigate_up=<widget name>]
	[navigate_down=<widget name>]
	[navigate_left=<widget name>]
	[navigate_right=<widget name>]
	[border.image.path=<path>] [border.image.top-left=<filename>] [border.image.top=<filename>] [border.image.top-right=<filename>] [border.image.right=<filename>] [border.image.bottom-right=<filename>] [border.image.bottom=<filename>] [border.image.bottom-left=<filename>] [border.image.left=<filename>]
	[border.selimage.path=<path>] [border.selimage.top-left=<filename>] [border.selimage.top=<filename>] [border.selimage.top-right=<filename>] [border.selimage.right=<filename>] [border.selimage.bottom-right=<filename>] [border.selimage.bottom=<filename>] [border.selimage.bottom-left=<filename>] [border.selimage.left=<filename>]
	[border.color=<color>] | [[border.color.r=<red>] [border.color.g=<green>] [border.color.b=<blue>] [border.color.a=<alpha>]]
	[border.selcolor=<color>] | [[border.selcolor.r=<red>] [border.selcolor.g=<green>] [border.selcolor.b=<blue>] [border.selcolor.a=<alpha>]]
	[border.thickness=<pixel>]
	[border.margin=<pixel>]
	[border.rcorners=<bool>]
	[font.path=<path>]
	[font.name=<filename>]
	[font.size=<pixel>]
	[item_width=<size>]
	[item_height=<size>]
	[item_hmargin=<pixel>]
	[item_vmargin=<pixel>]
	[cols=<columns>]
	[fixed_pos=<position>]
	[hloop=<bool>]
	[vloop=<bool>]
	[dim_items=<dimval>]
	[dim_top=<dimval>]
	[dim_bottom=<dimval>]
	[dim_left=<dimval>]
	[dim_right=<dimval>]
	[trans_items=<transval>]
	[trans_top=<transval>]
	[trans_bottom=<transval>]
	[trans_left=<transval>]
	[trans_right=<transval>]
>

Attributes

Widget Attributes Description
name Specifies the name of the widget. Default is "". The name of a widget cannot be set in a theme.
size The size attribute does only work if the element is a child of a <hbox> or <vbox>. Dependent on this, the size specifies either the width or the height of the widget. Per default the size is not set. See Size Values. The size of a widget cannot be set in a theme.
class Specifies the name of a defined theme class (see <mmstheme>).
bgcolor Specifies the background color. Per default the background color is not set. See Setting colors.
selbgcolor Specifies the background color which will used if the element is selected. Per default the selected background color is not set. See Setting colors.
bgimage Specifies an image file for the background. Per default the background image is not set. See Setting Images.
selbgimage Specifies a image file for the background which will used if the element is selected. Per default the selected background image is not set. See Setting Images.
margin Specifies the outer margin between the border and the next element. Unit is pixel. Default is "0".
focusable Specifies if the element is focusable. Values are "true" or "false". The default is element-specific. If this default is "false" then the value cannot be switched to "true".
selectable Specifies if the element is selectable. Values are "true" or "false". The default is element-specific. If this default is "false" then the value cannot be switched to "true".
up_arrow If widgets content can be scrolled you are able to set an 'up-arrow-widget' for it. Normally it is dependent on your layout. Here you have to specify the name of the 'up-arrow-widget'. Per default it is not set.
down_arrow If widgets content can be scrolled you are able to set a 'down-arrow-widget' for it. Normally it is dependent on your layout. Here you have to specify the name of the 'down-arrow-widget'. Per default it is not set.
left_arrow If widgets content can be scrolled you are able to set a 'left-arrow-widget' for it. Normally it is dependent on your layout. Here you have to specify the name of the 'left-arrow-widget'. Per default it is not set.
right_arrow If widgets content can be scrolled you are able to set a 'right-arrow-widget' for it. Normally it is dependent on your layout. Here you have to specify the name of the 'right-arrow-widget'. Per default it is not set.
data It is possible to store a data string for a widget which can be interpreted in your plugin. Per default the data string is empty.
navigate_up Use this parameter if you need another navigation then the default one. Here you can specify the name of the widget to which should navigate if the UP key is pressed. Per default it is not set and the default navigation will be used.
navigate_down Use this parameter if you need another navigation then the default one. Here you can specify the name of the widget to which should navigate if the DOWN key is pressed. Per default it is not set and the default navigation will be used.
navigate_left Use this parameter if you need another navigation then the default one. Here you can specify the name of the widget to which should navigate if the LEFT key is pressed. Per default it is not set and the default navigation will be used.
navigate_right Use this parameter if you need another navigation then the default one. Here you can specify the name of the widget to which should navigate if the RIGHT key is pressed. Per default it is not set and the default navigation will be used.
Border Attributes Description
border.image.path Here you can set the path of the border images. Default is "".
border.image.top-left Specifies the border image at the top-left corner. Default is "".
border.image.top Specifies the border image at the top. Default is "".
border.image.top-right Specifies the border image at the top-right corner. Default is "".
border.image.right Specifies the border image on the right side. Default is "".
border.image.bottom-right Specifies the border image at the bottom-right corner. Default is "".
border.image.bottom Specifies the border image at the bottom. Default is "".
border.image.bottom-left Specifies the border image at the bottom-left corner. Default is "".
border.image.left Specifies the border image on the left side. Default is "".
border.selimage.path Here you can set the path of the border images for selected items. Default is "".
border.selimage.top-left Specifies the border image for selected items at the top-left corner. Default is "".
border.selimage.top Specifies the border image for selected items at the top. Default is "".
border.selimage.top-right Specifies the border image for selected items at the top-right corner. Default is "".
border.selimage.right Specifies the border image for selected items on the right side. Default is "".
border.selimage.bottom-right Specifies the border image for selected items at the bottom-right corner. Default is "".
border.selimage.bottom Specifies the border image for selected items at the bottom. Default is "".
border.selimage.bottom-left Specifies the border image for selected items at the bottom-left corner. Default is "".
border.selimage.left Specifies the border image for selected items on the left side. Default is "".
border.color Specifies the border color. Per default the border color is not set. See Setting Colors.
border.selcolor Specifies the border color for selected items. Per default the selected border color is not set. See Setting Colors.
border.thickness Specifies the thickness of the border. Unit is pixel. Default is "0".
border.margin Specifies the inner margin between the border and the background of the element. Unit is pixel. Default is "0".
border.rcorners Values are "true" or "false". If you specify "true" round corners will be drawn. Default is "false".
Font Attributes Description
font.path Here you can set the path to the font. Default is "".
font.name Specifies the name of the file which contains the font. Default is "".
font.size Specifies the size of the text. Unit is pixel. Default is "22".
Menu Attributes Description
item_width Specifies the width of an item in the menu. Default is "100%". See Size Values.
item_height Specifies the height of an item in the menu. Default is "100%". See Size Values.
item_hmargin You can set an additional horizontal margin for each item. Unit is pixel. Default is "0".
item_vmargin You can set an additional vertical margin for each item. Unit is pixel. Default is "0".
cols The number of visible columns will be calculated based on the item_width attribute. If you want to have a specific number of columns (visible and invisible) you should set this cols attribute. Per default it is set to "0". In this case the menu uses only visible columns.
fixed_pos You can set a fixed position of the selected item. This works correct only for menus with cols="1" (vertical menu with single column) or menus with cols="9999" (horizontal menu with single row). The position 0 is the first item. If you set a value greater or equal than the visible items, the fixed selected item is in the middle. If you do not want a fixed position you should use fixed_pos="-1". This is default.
hloop If set to "true", you can press LEFT/RIGHT keys and the menu items will rotate horizontally without jumping out of the menu. The default is "false".
vloop If set to "true", you can press UP/DOWN keys and the menu items will rotate vertically without jumping out of the menu. The default is "false".
dim_items It is possible to dim out all menu items if the menu has not the focus. Values between "0" and "255" are supported. The value "255" means maximum dim-out. Default is "0".
dim_top Dim out items at the top of the menu. This works only for menus with fixed selection (see fixed_pos attribute). Values between "0" and "255" are supported. The value "255" means maximum dim-out. Default is "0".
dim_bottom Dim out items at the bottom of the menu. This works only for menus with fixed selection (see fixed_pos attribute). Values between "0" and "255" are supported. The value "255" means maximum dim-out. Default is "0".
dim_left Dim out items on the left of the menu. This works only for menus with fixed selection (see fixed_pos attribute). Values between "0" and "255" are supported. The value "255" means maximum dim-out. Default is "0".
dim_right Dim out items on the right of the menu. This works only for menus with fixed selection (see fixed_pos attribute). Values between "0" and "255" are supported. The value "255" means maximum dim-out. Default is "0".
trans_items It is possible to make all menu items transparent if the menu has not the focus. Values between "0" and "255" are supported. The value "255" means maximum transparency. Default is "0".
trans_top Items at the top of the menu can be made transparent. This works only for menus with fixed selection (see fixed_pos attribute). Values between "0" and "255" are supported. The value "255" means maximum transparency. Default is "0".
trans_bottom Items at the bottom of the menu can be made transparent. This works only for menus with fixed selection (see fixed_pos attribute). Values between "0" and "255" are supported. The value "255" means maximum transparency. Default is "0".
trans_left Items on the left of the menu can be made transparent. This works only for menus with fixed selection (see fixed_pos attribute). Values between "0" and "255" are supported. The value "255" means maximum transparency. Default is "0".
trans_right Items on the right of the menu can be made transparent. This works only for menus with fixed selection (see fixed_pos attribute). Values between "0" and "255" are supported. The value "255" means maximum transparency. Default is "0".

Setting colors

There are two ways to set a color. For example the color attribute is named color.

First way is: color="#rrggbbaa". There is a hash followed by hexadecimal byte values for red, green, blue and alpha.
If you set color="#ff0000ff" you use red which is not transparent.
If you set color="#00000000" the color is marked as not set.

Second way is to set the four color values separately: color.r=<red> color.g=<green> color.b=<blue> color.a=<alpha>. Each value has range from 0 to 255 decimal.
If you set color.r="0" color.g="0" color.b="255" color.a="255" you use blue which is not transparent.
If you set color.a="0" the color is marked as not set.

Note on alpha value:
The value 0 (00) means that the whole color is not set. The value 255 (ff) means that the color is set with full opacity. The values 1..254 (01..fe) sets a transparent color.

Setting images

There are two ways to set a image. For example the image attribute is named image.

First way is: image="path/filename". You know the complete path and the filename (inclusive the extension) to the image and put it directly to the image attribute. If you set image="" the image is marked as not set.

Second way is to set the path and the filename separately: image.path="path" image.name="filename". If you set image.path="" the image path is marked as not set. If you set image.name="" the image name is marked as not set.

Note on transparency of images:
You should only use images without alpha channel. Because currently we cannot handle image-based transparency. But if you want to have a transparent image you can use the alpha value of the correspondent color. For example you can load the bgimage="<filename>" of a window or a widget and set the bgcolor="#00000080". The alpha value 80 hex sets the half opacity of the bgimage.

Set an item template

The specific feature of the menu widget is that you can dynamically add and delete items. The layout of these items is free. Free means that you can use all widgets listed here (except <menu> itself) to build your own item template.

One requirement to the item template: The top-level widget must be selectable.

The examples below demonstrates how you can build different kind of menus.

Remarks

This element can be used as child within windows and widgets.

If you use <menuitem> the dialogmanager can add one or more menu items. For fixed menus you do not need to add it with usage of the C++ API.

Examples

Simple Listbox

The dialog manager creates a <mainwindow> with one <menu> widget named listbox. The item template of this menu consists of only one <label> widget as top-level widget. 
<mmsdialog>
    <mainwindow w="320" h="320">
        <menu name="listbox" item_height="32px">
            <label margin="1" font.size="22px"/>
        </menu>
    </mainwindow>
</mmsdialog>

Listbox with image

The dialog manager creates a <mainwindow> with one <menu> widget with the name listbox. The item template of this menu consists of a <button> widget as top-level widget. This <button> has a <hbox> as a child. And the <hbox> includes an <image> and a <label>.

In your plugin you can access the <image> of each item through the name listbox_image and the <label> through the name listbox_label. 
<mmsdialog>
    <mainwindow w="320" h="320">
        <menu name="listbox" item_height="32px">
            <button margin="1">
                <hbox>
                    <image name="listbox_image" size="30px"/>
                    <label name="listbox_label" font.size="22px"/>
                </hbox>
            </button>
        </menu>
    </mainwindow>
</mmsdialog>

Image Menu

The dialog manager creates a <mainwindow> with one <menu> widget with the name images. The item template of this menu consists of a <button> widget as top-level widget. This <button> has a <vbox> as a child. And so on...
This menu has two columns and two visible rows (50% each item). Each item displays an image and a text below it.
In your plugin you can access the <image> of each item through the name item_image and the <textbox> through the name <item_textbox. 
<mmsdialog>
    <mainwindow w="320" h="320">
        <menu name="images" item_height="50%" item_height="50%">
            <button margin="10"/>
                <vbox>
                    <hbox>
                        <hbox size="20%"/>
                        <image name="item_image"/>
                        <hbox size="20%"/>
                    </hbox>
                    <textbox name="item_textbox" size="44px" font.size="22px"/>
                </vbox>
            </button>
        </menu>
    </mainwindow>
</mmsdialog>

Image menu using <mmstheme>

This example demonstrates how to define a complete menu layout within <mmstheme> and using it within a dialog.

The dialog manager creates a <mainwindow> with one <menu> widget with the name images and the theme class image_menu. Each item displays an image and a text below. In your plugin you can access the <image> of each item over the name item_image and the <textbox> over the name item_textbox. 
<mmsdialog>
    <mainwindow w="320" h="320">
        <menu name="images" class="image_menu"/>
    </mainwindow>
</mmsdialog>
The item template of this menu consists of a <button> widget as top-level widget. This <button> has a <vbox> as child. And so on...
This menu has two columns and two visible rows (50% each item). 
<mmstheme>
    <class name="image_menu" type="menu" item_height="50%" item_height="50%">
        <button margin="10"/>
            <vbox>
                <hbox>
                    <hbox size="20%"/>
                    <image name="item_image"/>
                    <hbox size="20%"/>
                </hbox>
                <textbox name="item_textbox" size="44px" font.size="22px"/>
            </vbox>
        </button>
    </class>
</mmstheme>

Create and fill a simple listbox

The dialog manager creates a <mainwindow> with one <menu> widget with the name listbox. The item template of this menu consists of only one <label> widget as top-level widget (named mylabel). After creation three items will be added to the listbox by the dialogmanager. 
<mmsdialog>
    <mainwindow w="320" h="320">
        <menu name="listbox" item_height="32px">
            <label name="mylabel" margin="1" font.size="22px"/>
            <menuitem widget.mylabel.text="First item"/>
            <menuitem widget.mylabel.text="Second item"/>
            <menuitem widget.mylabel.text="Third item"/>
        </menu>
    </mainwindow>
</mmsdialog>
